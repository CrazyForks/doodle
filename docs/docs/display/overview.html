The Display

An app's root container​

The Display holds an app's View hierarchy, and behaves like a basic container. It is not a View however, so many of the capabilities of Views are not available for the Display.

tip

Layouts can be applied to the Display as well.

The Display is available for injection by default.

package display import io.nacular.doodle.application.Application import io.nacular.doodle.application.application import io.nacular.doodle.core.Display import org.kodein.di.instance //sampleStart class MyApp(display: Display): Application { init { // ... } override fun shutdown() {} } fun main() { application { MyApp(display = instance()) } } //sampleEnd

Adding Views to the Display​

The Display has a children property that contains all its direct descendants. These top-level views have no parent and are the top-most ancestors of all other Views in an app. An app can have any number of these Views.

caution

Top-level Views are displayed, yet they have no parent. Similarly, Views can have parents even before they are in the display hierarchy: when the parents themselves are not displayed. This means the parent property says nothing about a View being in the Display hierarchy. Luckily, View has the displayed property for this exact purpose.

Add a top-level View like this.

package display import io.nacular.doodle.application.Application import io.nacular.doodle.core.Display import io.nacular.doodle.core.view //sampleStart class AddViewApp(display: Display): Application { init { val view = view {} display += view } override fun shutdown() { /*...*/ } } //sampleEnd

And remove it like this.

package display import io.nacular.doodle.application.Application import io.nacular.doodle.core.Display import io.nacular.doodle.core.view //sampleStart class RemoveViewApp(display: Display): Application { init { val view = view {} display -= view } override fun shutdown() { /*...*/ } } //sampleEnd

Launch mode changes the Display​

A Stand-Alone Web app that uses the entire page will have a Display tied to the page body. While one hosted in an element will have a Display that is tied to that element.

This is transparent to the app.

package display import io.nacular.doodle.application.application import org.kodein.di.instance fun topLevelLaunch() { //sampleStart application(modules = listOf(/*...*/)) { MyApp(display = instance() /*,...*/) } //sampleEnd }

Here the Display will be tied to someDiv.

package display import io.nacular.doodle.application.application import org.kodein.di.instance import org.w3c.dom.HTMLElement fun embeddedLaunch(element: HTMLElement) { //sampleStart application(element, modules = listOf(/*...*/)) { MyApp(display = instance() /*,...*/) } //sampleEnd }

The Displays for a Nested app sits within the View hosting it. This means changes to that View's size will change the Display size.
