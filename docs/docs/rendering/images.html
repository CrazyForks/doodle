Images

Doodle allows you to load Images into your app for rendering using the ImageLoader.

The following app tries to load and render an image. Notice that ImageLoader.load returns Image?, which is null when the image fails to load for some reason. Notice the load method is suspending, so it much be called from another suspend method or from a CoroutineScope.

package rendering import io.nacular.doodle.application.Application import io.nacular.doodle.controls.Photo import io.nacular.doodle.core.Display import io.nacular.doodle.image.Image import io.nacular.doodle.image.ImageLoader import kotlinx.coroutines.CoroutineScope import kotlinx.coroutines.launch //sampleStart class SimpleImageApp(display: Display, imageLoader: ImageLoader, scope: CoroutineScope): Application { init { scope.launch { val image: Image? = imageLoader.load("some_image_url") // won't get here until load resolves image?.let { display += Photo(it) } } } override fun shutdown() {} } //sampleEnd
Module Required

You must include the ImageModule in your application in order to use these features.

package rendering import io.nacular.doodle.application.Application import io.nacular.doodle.application.Modules.Companion.ImageModule import io.nacular.doodle.application.application import io.nacular.doodle.core.Display import io.nacular.doodle.image.ImageLoader import org.kodein.di.instance class ImageLoaderApp(display: Display, images: ImageLoader): Application { override fun shutdown() {} } fun imageLoader() { //sampleStart application(modules = listOf(ImageModule)) { ImageLoaderApp(display = instance(), images = instance()) } //sampleEnd }

Doodle uses opt-in modules like this to improve bundle size.

Draw images onto Canvas​

The method for doing this allows you to customize how the image drawing by specify the following.

  • Source rectangular in the image
  • Destination rectangle to draw into
  • Opacity to draw the image with
  • Radius for the image corners
